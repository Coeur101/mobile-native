## Why

当前登录页虽然已经支持邮箱注册、验证码登录、密码登录和密码重置，但界面把“模式切换”“主操作”“次要入口”几乎都做成同权重按钮并堆在同一块区域里，导致信息层级混乱、交互节奏单一、移动端缺少引导感与反馈感。认证能力已经具备可用基础，现在需要先把登录页 UI 体验重做清楚，否则继续叠加功能只会让入口更拥挤。

## What Changes

- 重构登录页的信息架构，拆开“入口选择”“表单输入”“主 CTA”“辅助操作”四个层级，避免所有动作都挤在同一按钮矩阵中。
- 将当前认证首页改造成更有引导感的移动端交互流程，强化首屏视觉焦点、状态切换反馈和步骤推进感。
- 重新设计登录 / 注册 / 找回密码的切换方式，让模式切换更像导航或分段选择，而不是与提交按钮混排。
- 为验证码登录、密码登录、注册、重置密码分别定义更清晰的页面状态、过渡反馈和文案层次。
- 补充登录页 UI 自动化验证，确保新的交互结构在移动端视口下稳定可用。
- 不修改已有 Supabase 认证能力、不新增微信登录、不改变当前认证业务规则。

## Capabilities

### New Capabilities
- `auth-entry-ui`: 规范移动端认证入口页的视觉层级、状态切换、主次操作布局与反馈体验。

### Modified Capabilities
- `user-auth`: 收紧移动端认证流的交互要求，明确模式切换不能与提交动作混排，且每一步必须有清晰的主操作与辅助操作层级。

## Impact

- 主要影响代码与设计实现：
  - `src/pages/login/LoginPage.tsx`
  - 可能涉及认证页复用的 UI 组件、动画参数与文案组织
  - Playwright 登录页 UI 测试
- 不影响：
  - `src/services/auth/*` 的核心认证接口
  - Supabase 配置与会话持久化规则
  - 当前 7 天登录态、邮箱验证码、密码登录与密码重置能力本身
